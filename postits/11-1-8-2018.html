I started learning how to use the JavaScript library jQuery a couple days ago. It's a mature package that allows a programmer to add interactivity to webpages, respond to user interactions, make AJAX requests and make DOM programming easier all without being verbose. The motto is even something like, Do more, write less. It's popularity grew in the early 2000's when browsers were less standardised. jQuery is used to access the DOM (Document Object Model) by selecting CSS selectors to select HTML elements and do something with them.

Let's do a small alert piece with jQuery. Assume this is linked to an HTML script that has a red button. This alert message is first hidden from view, then it slides down in one second, remains to be red for three seconds and then disappears from view.

$('#alert').hide();
$('#alert').slideDown(1000); //written in milleseconds
$('#alert').delay(3000);
$('#alert').slideUp

Which can also be written as:

$('#alert').hide().slideDown(1000).delay(3000).slideUp

The latter is actually a better way to write this because it avoids having to find and select #alert every time and will save a few bits as well because theer are less pointers and returns being called for each #alert selection.
