If you're familiar with JavaScript, you've likely used or written JSON files for AJAx purposes. GeoJSONs are similar to JSON, but it's purposefully for encoding geographic data structures through specific geometry types: Point, LineString, Polygon, Multipoint, MultiLineString and MultiPolygon. It also supports non-spatial elements of geographic data with Feature and sets of features called FeatureCollection. It can be used by various mapping software and GIS packages.
    Point and MultiPoint are used for addresses and exact locations.
    {"type": "Point", "coordinates": [10,10]} {"type": "MultiPoint", "coordinates": [ [0, 0], [5, 5], [10,10]}
    LineString and MultiLineString are used for streets, interstates and boundaries.
    {"type": "LineString", "coordiantes": [ [0,0], [1, 1], [2, 2] ]} {"type": "MultiLineString", "coordiantes": [ [[0,0], [1, 1]], [[2, 2], [3, 3]] ]}
    Polygon and MultiPolyon are used for politically bounded surface areas, land types or land formations. Polygon can also be used as an interrelated set. An example would be if you were mapping the ocean, but when you come across an island you wouldn't map that as water.

    { "type": "Polygon", "coordinates": [ [[0, 0], [0, 2], [2, 2], [2, 0] ]]}
    { "type": "Polygon", "cooridinates": [ [[35, 10], [45, 45], [15, 40], [10, 20], [35, 10]], [[20, 30], [35, 35], [30, 20], [20, 30]] ]}
    { "type": "MultiPolygon", "coordinates": [ [ [[30, 20], [45, 40], [10, 40], [30, 20]] ], [ [[15, 5], [40, 10], [10, 20], [5, 10], [15, 5]] ] ] }
