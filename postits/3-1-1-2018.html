I started with the course Computer Science 61A: The Structure and Interpretation of Computer Programs. It's the introductory course to the CS major at Cal that introduces major topics and concepts in computer science. It's mostly in Python, with bits of Scheme and SQL. The first tool I started to use was Python3. Python is an object-oriented program. I like to think of actual objects with specific attributes that can be associated with it to best contextualise what's going on in the program. For example, bank accounts are a great way to think about it, but more on that in another post. Aside from function definition, the first thing reviewed was Environment Diagrams. These are particularily helpful to understand what you're code is doing and what happens during each part of the code. The initial function or functions comprise the Global or Parent Frame. That function binds to the function value in that current frame. For example if you had:
def add_one(x):
y = x + 1

return y
then your global frame would have add pointing to func add(x, y) and have a place value for x, y and the return value. However, if you have multiple functions, then the Global Frame will be all the functions pointing to their function, and the subsequent functions will be the specifics of the functions. This can be a little difficult to understand without visuals, so I encourage this site to visualise what's going on with programs. It's definitly a difficult thing to do at first and even a bit tedious to draw and redraw, but it's worth udnerstanding and working through.
