Going off of yesterday's post, the coding community can be very welcoming to young programmers. We talked briefly about weird things called npm packages as a black box that can basically do things. Let's delve a little deeper into what these are exactly. Npm packages contain pre-existing code, so you don't have reinvent the wheel so to speak. A good starting point to see what's available to you is the npm website. (An account is free to make if necessary). Consider the following when looking for a package:
    Popularity. While popularity isn't everything, it is a good indicator of broad usage, frequent testing and scaffolding to help you learn it.
    Version Number and Number of Releases.The more releases, the more active. It's also important to view the date and frequency of intervals. Under the name, you can also see if the package is passing the tests.
    Github. Sometimes the Github repo can be more popular than the npm website. Also, the owner might update the repo more than the npm site, so watch out for that.
This npm package called http-server is a great tool to understand, test and deploy locally. It has easy (zero) set up and can be run on the command line. To install, open the terminal and write npm install http-server -g. The -g flag means that you're installing it globally and giving it the ability to be run from the command line. To run, you'll first create an index.html file inside of a folder. You may fill your static web page with whatever you like. Once you have that saved in a folder, type http-server [folder name]. It should print out a message like this:

Starting up http-server, serving [folder_name]/
Available on:
http://127.0.0.1:8080
http://10.103.160.249:8080
Hit CTRL-C to stop the server

What do all these numbers mean? The first set 127.0.0.1 is a default IP address on all computers. The 8080 is the default port. You can flag -p to adjust the port.

Hope this helped as a starting off point to npm!
