Continuing in the same script as the one you have for the game state, let's now create a start and end of the game. To begin first ask yourself if starting and ending game functionality should be in the Start or Update function. Once you have that determined, ask how you will oscillate between the two. Before you write the methods for starting and ending the game, list out all of the differences between the two and what functionality you want them to have. For starting the game, it might be useful to allow movements. For ending the game, it would be useful to notify the player the game is over and disable further movements.

We decided earlier that the game will begin once a player presses the spacebar. This can be tricky. So, I recommend when calling the method that will start the game, add this conditional by typing the following: Input.GetKeyUp (KeyCode.Space). This code is essentially looking to see if the spacebar has been pressed down and has come back up. This is significant because it will only take one key press to start the game, and ignore further presses the player may incur. Before scripting a function to end the game, list out the functionality you think you'll need. I recommend considering a way to restart the game (instead of having to restart the whole program!).

Audio is quite complex to explain without visuals or audio samples, so my best recommendation is to tell you to put all the sounds in the same mixer and test them out to your own pleasing.
